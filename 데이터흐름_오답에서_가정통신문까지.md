# 가경T 과제 검사 시스템 — 데이터 흐름 (오답 입력 → 진단 매칭 → 가정통신문)

> 이 문서는 GPT 등 외부 AI에게 시스템 구조를 설명하기 위한 참고용입니다.
> "학생이 오답을 입력하는 순간부터, 그 오답이 어떤 정보와 매칭되어, 최종적으로 가정통신문 PDF가 만들어지기까지"의 전체 파이프라인을 정리합니다.
>
> - 학생용: `index.html` (마킹/제출)  · 강사용: `dashboard.html` (집계/리포트/문자/가정통신문)
> - 백엔드: Google Apps Script(`AppsScript.gs`) + Google Sheets  · 배포: GitHub Pages(repo `oreum1222/homework`)
> - 정적 데이터: `data/hw-<courseId>-w<n>.json`(문항 진단표), `data/observ-dict.json`(관찰문장 사전), `data/courses.js`(수업·주차 레지스트리)

---

## 0. 한눈에 보는 파이프라인

```
[문항 진단표]  data/hw-*.json  (문항번호 → 영역/유형/핵심요소/처방)
      │  (학생이 푼 회차의 문항 메타 제공)
      ▼
[학생 마킹]  index.html  ──제출──▶  Apps Script doPost  ──▶  Google Sheet(제출 시트)
   틀린 문항번호 체크          각 오답을 진단표와 매칭해
   + 학습진단 설문(과제해결정도)   wrongDetailsJson 으로 저장
      │
      ▼
[집계]  dashboard.html  ◀──action=list──  Apps Script doGet
   buildReports() 가 제출들을 학생별로 묶어
   REP.master[] (학생별 누적 진단 객체) 생성
      │
      ▼
[가정통신문 작성 탭]
   ① 학생 선택 → nlFindMaster() 로 그 학생 REP.master 찾기
   ② 1페이지: REP.master → 회차별/단원별 정답률·진단 (그래프)
   ③ 관찰문장 사전(observ-dict.json) 체크 → nlAssemble() 로 종합 의견 초안
   ④ 2페이지: 종합 의견(≤900자)
      │
      ▼
[PDF]  nlMakePdf()  →  2페이지 A4 (1p 데이터 / 2p 코멘트)
```

핵심 매칭 키는 **문항번호(`no`)** 와 **학생 이름(정규화)** 두 가지입니다.
- 오답↔진단정보 매칭: `제출.wrong[] 의 no` ↔ `hw-*.json questions[].no`
- 학생↔리포트↔가정통신문 매칭: `normName(이름)` (대괄호·공백 무시: `[실험용]`=`실험용`)

## 1. 문항 진단표 — `data/hw-<courseId>-w<n>.json`

회차(주차)마다 한 파일. "문항번호가 어떤 영역·유형이고, 틀리면 무엇을 보강해야 하는지"의 사전.

```jsonc
{
  "courseId": "summer-grammar",
  "week": 1,                          // 주차값(고3 월별은 합성값 (월index+1)*100+주차, 예: 6월2주=102)
  "weekMeta": {
    "label": "1주차 · 품사편",
    "area":  "품사",                  // ★ 이 회차의 '단원(대영역)' — 가정통신문 단원별 그래프의 기준
    "desc":  "..."
  },
  "scoring": { "passRate": 80 },
  "sections": [                       // main(본 시험) + sat(수능) 등으로 분리 가능
    {
      "key": "main", "title": "품사편",
      "questions": [
        {
          "no": 1,                    // ★ 매칭 키: 학생이 체크한 오답 번호와 연결됨
          "id": "P1",
          "area": "품사 구별·통용",    // ★ 세부 채점영역(문항 단위, 단원보다 잘게 쪼갬)
          "type": "",                 // 세부 유형(소분류)
          "source": "",               // 기출 출처(예: "2015년 고1 3월")
          "coreElement": "'같이'의 품사 구별(조사/부사)",   // 이 문항이 묻는 핵심
          "weakness1": "…틀렸다면 …개념이 흔들린다는 신호예요.",  // 틀렸을 때 진단 멘트
          "weakness2": "",
          "remediation": "체언 뒤 '~와 함께'=조사, 용언 수식=부사",  // ★ 보강 처방
          "parentNote": "",           // (선택) 학부모용 코멘트
          "three": false, "wr": null, "in_typebook": false, "note": ""
        }
      ]
    }
  ],
  "studyCheck": [                     // 제출 시 '학습 진단 설문'(과제 수행 정도 등)
    { "num": 1, "tag": "done", "q": "이번 과제, 어디까지 했나요?",
      "opts": ["전부 끝냄","대부분(70% 이상)","절반쯤","조금만(30% 이하)","아직 시작 못 함"] },
    { "num": 2, "tag": "dueDate", "type": "date", "q": "완수 예정일", "showIf": {"q":1,"minIndex":1} }
    // … duePledge 등
  ],
  "studyDiagnosticCodes": [ /* 학습습관 진단 코드 매핑 */ ]
}
```

**중요한 구분 (가정통신문 설계의 핵심)**
- `weekMeta.area` = **단원(대영역)**: 품사 / 문장성분 / 음운 체계 / 음운 변동 … → 부모용 "단원별 정답률"의 기준.
- `questions[].area` = **세부 채점영역**(문항 단위): 품사 구별·통용 / 띄어쓰기·맞춤법 / 자음군 단순화 … → 너무 잘게 쪼개져(한 학생 18개+) **차트로 쓰면 난잡**. → 텍스트 진단(가장 약한 영역 Top)으로만 사용.

## 2. 학생 오답 입력 → 제출 (`index.html` → Apps Script → Sheet)

1. 학생이 수업·주차 선택 → 해당 `hw-*.json` 로드.
2. **STEP1 학습 상태**: "숙제 하였습니다/안 하였습니다" + 회차별 '안 풂' 토글.
   - 안 함(≤10%)이면 **각서 페이지**로(반성문). `과제해결정도(%)` 산출.
3. **STEP2 오답 마킹**: 틀린 문항번호를 그리드에서 체크.
4. 제출 시, 클라이언트가 각 오답 `no`를 `hw-*.json`의 `questions[no]`와 매칭하여 **`wrongDetailsJson`** 을 만들어 함께 전송.

**제출 1건(시트 1행 = action=list 의 한 객체) 주요 필드**
```jsonc
{
  "timestamp": "2026-06-03T03:27:10Z",
  "courseId": "summer-grammar",
  "name": "김람쥐",
  "week": 1, "weekLabel": "1주차 · 품사편",
  "area": "품사",                 // weekMeta.area (이 회차 단원)
  "정답률": 82,                   // 맞은 문항 기준 %
  "과제해결정도": 100,            // 과제 수행 정도 %(분수로 저장된 옛 데이터는 ×100 환산)
  "완수상태": "전부 끝냄",
  "wrongDetailsJson": "[ {오답상세}, … ]"   // ↓ 아래 구조 (문자열화된 JSON)
}
```

**`wrongDetailsJson` 의 한 원소 = 오답 1개 (진단표에서 복사된 매칭 정보)**
```jsonc
{
  "no": 10,                       // 틀린 문항번호 (hw-*.json questions[].no 와 동일)
  "area": "탈락",                 // 세부 채점영역
  "type": "자음군 단순화",        // 세부 유형
  "source": "2015년 고1 3월",     // 기출 출처
  "coreElement": "자음군 단순화 파악",
  "weakness1": "‘자음군 단순화’ 적용 문항 … 신호예요.",
  "weakness2": "",
  "remediation": "자음군 단순화·ㄹ/ㅎ/ㅡ 탈락 — 환경별로 정리해 보세요.",
  "parentNote": "", "three": false, "wr": null
}
```
> 즉 **오답이 들어오는 순간, 진단표의 메타(area/type/source/coreElement/weakness/remediation)가 그 오답에 박제되어 저장**됩니다. 이후 집계·리포트·가정통신문은 이 박제된 정보를 읽기만 하면 됩니다.

## 3. 집계 — `dashboard.html buildReports()` → `REP.master[]`

`action=list`로 전체 제출을 받아(`ALL_DATA`), **학생별로 묶어** 누적 진단 객체 `REP.master[]`를 만듭니다. (실험용/테스트/제외명단 계정은 빼고)

**학생 1명의 master 객체 (가정통신문이 읽는 핵심 데이터)**
```jsonc
{
  "name": "김람쥐", "courseId": "summer-grammar",
  "n": 4,                          // 제출 회차 수
  "seq":   [1, 2, 7, 8],           // 회차(주차)들
  "rates": [82, 80, 90, 90],       // 회차별 정답률 → '회차별 정답률 막대'
  "avgRate": 85, "trend": "유지",  // 평균·추세(상승/유지/하락)
  "totalWrong": 36,                // 누적 오답 수
  "perWeek": [                     // 회차별 상세 → '단원별 정답률' 집계에 사용
    { "week": 1, "area": "품사",       "rate": 82, "wrongN": 9,  "wrongs": [1,6,…], "areaStat": {…} },
    { "week": 2, "area": "문장성분",   "rate": 80, "wrongN": 10, … },
    { "week": 7, "area": "음운 체계",  "rate": 90, "wrongN": 5,  … },
    { "week": 8, "area": "음운 변동",  "rate": 90, "wrongN": 12, … }
  ],
  "areaAgg": {                     // ★ 세부 채점영역(questions[].area) 누적 — 잘게 쪼개짐
    "품사 구별·통용": {"total":60,"wrong":3},
    "띄어쓰기·맞춤법": {"total":56,"wrong":6},
    "음운 변동(종합)": {"total":55,"wrong":11}, …  // 한 학생 18개+ 가능 → 차트 부적합
  },
  "recurring": [ {"name":"음운 변동(종합)","cnt":2} ],  // 여러 회차 반복 약점
  "strengths": [ {"name":"음운 체계","diag":"정답률 90%…"} ],
  "typeTop":   [ {"t":"자음군 단순화","c":2}, … ]       // 세부 유형 빈도 상위
}
```

집계 매칭 규칙
- `wrongDetailsJson[].area` 를 누적 → `areaAgg`(세부영역별 total/wrong).
- 제출의 `area`(=weekMeta.area) + `정답률` + `오답수` → `perWeek`(단원별).
- 이름은 `normName_`(대괄호·공백 제거)으로 동일인 병합.

## 4. 가정통신문 생성 — `dashboard.html` 「가정통신문 작성」 탭

### 4-1. 학생 선택
`nlFindMaster(이름)` → REP.master에서 해당 학생 객체. (명단 roster + master에서 학생 목록 구성)

### 4-2. 1페이지 — 데이터 면 (`nlBuildPage1(name, m)`)
master(`m`)만으로 구성:
- **스탯 4칸**: 평균 정답률 / 추세 / 제출 회차 / 누적 오답 — `m.avgRate, m.trend, m.n, m.totalWrong`
- **회차별 정답률 막대** (`nlBarSVG`): `m.seq` × `m.rates`, x축 라벨은 `perWeek.area`(단원명)
- **단원별 정답률 가로막대** (`nlUnitBars_` ← `nlUnitAgg_`): `perWeek`를 `area`로 묶어 단원별 평균 정답률·오답수 (약한 순). ← **세부영역 도넛(18조각)을 대체**
- **학습 진단 카드**: 보완점(`rIntegrated(m)` — areaAgg/recurring/typeTop/추이를 문장화) + 강점(`m.strengths`)
  - 세부 채점영역은 여기서 **텍스트로만** 노출(차트로 안 쪼갬).

### 4-3. 종합 의견 초안 — 관찰문장 사전 (`data/observ-dict.json` → `nlAssemble`)
오답 데이터와 **별개로**, 교사가 수업/숙제/조교 관찰을 체크해서 만드는 정성 코멘트.
```jsonc
// observ-dict.json
{
  "categories": [ { "cat":"수업 중 관찰", "subs":[ { "sub":"자리와 태도",
      "items":[ { "kw":"앞자리", "interp":"수업을 가까이 …",
        "flow":"앞자리에 앉아 … 의지가 느껴집니다.",   // ★ 어미 다양화한 완결 문장(이어붙임용)
        "short":"…", "long":"…(상세 단락)", "remedy":"…(마무리 지도문장)",
        "theme":"앞자리" } ] } ] } ],   // theme: 중복 제거 키
  "cards": [ {"name":"앞자리에 앉는 학생","flow":"…","theme":"앞자리"} ],  // 빠른 문장카드
  "convert": [ {"from":"숙제를 안 해옴","to":"과제 루틴이 …","expand":"…"} ]  // 보완표현 변환
}
```
- 체크 → `nlAssemble()`가 `flow` 문장들을 **주어 1회 + 어미 다양 + 중복(theme) 제거**로 자연스럽게 엮음.
- 기본 = `flow`(요약), '긴 문장으로' 토글 = `long`(상세).
- **글자 수 제한 NL_MAX=900**(한 쪽). 항목 추가 시 초과하면 자동 차단, 초과 코멘트는 「900자 내로 정리」 버튼으로 정리.

### 4-4. 2페이지 — 종합 의견 면 (`nlBuildPage2`)
인사말 + 선생님 종합 의견(≤900자) + 서명(연구소·교사 드림). 서명은 흐름 배치(겹침 없음).

### 4-5. PDF (`nlMakePdf`)
- 고정 2페이지(각 A4 한 장 통째 캡처, html2canvas → jsPDF). 박스가 두 장에 걸쳐 잘리지 않음.
- 디자인: 아이보리(#f7f4ec) × 네이비(#1c2c52) × 골드(#b08d57), 남색 로고(`config.js LOGO_NAVY_URL`).

## 5. 매칭 관계 요약 (한 줄 정리)

| 단계 | 입력 | 매칭 키 | 출력 |
|---|---|---|---|
| 오답 저장 | 학생이 체크한 틀린 문항번호 | `no` ↔ `hw-*.json questions[].no` | `wrongDetailsJson`(area/type/source/coreElement/remediation 박제) |
| 집계 | 제출들(`action=list`) | `normName(이름)` | `REP.master[]` (seq/rates/perWeek/areaAgg/strengths/recurring/typeTop) |
| 1페이지 | master | `m.perWeek.area`(단원), `m.areaAgg`(세부) | 회차별·단원별 정답률 + 진단 텍스트 |
| 코멘트 | 교사 관찰 체크 | `theme`(중복제거) | `flow` 문장 엮은 종합 의견(≤900자) |
| PDF | 1p(데이터)+2p(코멘트) | 학생 이름 | 가정통신문 2페이지 A4 |

## 6. 용어/주의
- **단원(대영역)** `weekMeta.area` = 부모용 그래프 기준(품사/문장성분/음운 체계/음운 변동…).
- **세부 채점영역** `questions[].area` = 진단 정밀도용(너무 잘아서 차트 X, 텍스트 진단 O).
- 고3 월별 주차값은 합성값 `(월index+1)*100+주차`(6월 1~4주 = 101~104).
- 이름 매칭은 항상 `normName_`(대괄호·공백 무시). 실험용/테스트/제외명단 계정은 집계·발송에서 제외.
- 정답률·과제해결정도는 0~100. 옛 분수(0~1) 저장분은 ×100 환산.
