# 문자 발송 대시보드 기획서 (Solapi 연동)

> 가경T 숙제 검사 시스템 대시보드 **내부에 '문자 발송' 탭**을 추가한다.
> 명단 시트 연락처 + 제출 데이터를 활용해 **알림톡(우선) → 문자(폴백)** 로 보내고,
> 수동 발송 + 일부 자동(스케줄)까지 운영한다.

---

## 1. 결정 사항 (확정)
| 항목 | 결정 |
|---|---|
| 연락처 소스 | **명단(roster) 시트에 미리 등록** — 교사 관리, 이름+수업으로 제출 데이터와 매칭 |
| 발송 채널 | **카카오 알림톡 우선 → 실패 시 문자(LMS) 자동 대체** |
| 시나리오 | **체크박스로 선택** (미제출·미완 독려 / 가정통신문 요약 / 결석·지각 / 직접 작성) |
| 운영 방식 | **수동 버튼 발송 + 일부 자동 스케줄** |

---

## 2. 전체 아키텍처
```
[대시보드 '문자 발송' 탭]  (정적 · GitHub Pages)
        │  ① 명단/대상 조회 (GET action=roster)
        │  ② 발송 요청 (POST action=sendMessages, 수신자+메시지 배열)
        ▼
[Apps Script 백엔드]  ← API 키/시크릿은 여기 Script Properties에만 보관
        │  ③ HMAC 서명 후 Solapi REST 호출
        │  ④ 결과를 '발송로그' 시트에 기록
        ▼
[Solapi]  → 알림톡(pfId/template) → 실패 시 문자(LMS) 폴백 → 보호자 휴대폰
```
- **핵심 보안 원칙**: 정적 페이지에는 절대 API 키를 두지 않는다. 모든 발송은 Apps Script가 중계.
- 기존 숙제 백엔드(제출/조회)와 **같은 스프레드시트, 다른 시트 + 새 doPost action** 으로 확장.

## 3. 데이터 모델 (구글 시트)
### 3-1. `명단` 시트 (교사가 직접 관리)
| 열 | 설명 | 필수 |
|---|---|---|
| courseId | 수업 식별자 (예: go3-regular) | ✔ |
| name | 학생 이름 (제출 데이터와 정확히 일치) | ✔ |
| school | 학교 (동명이인 구분용) | |
| guardianName | 보호자 성함 | |
| guardianPhone | **보호자 연락처** (010xxxxxxxx) | ✔ |
| studentPhone | 학생 연락처 (선택) | |
| consent | 수신 동의 (Y/N) | ✔ |
| memo | 비고 | |

→ 이름(+학교)으로 제출 데이터(master)와 매칭. **번호 없거나 consent=N이면 발송 대상에서 자동 제외**(경고 표시).

### 3-2. `발송로그` 시트 (시스템 자동 기록)
`timestamp · 수업 · 이름 · 번호(마스킹) · 채널(알림톡/문자) · 시나리오 · 메시지 · 결과(성공/실패) · groupId/messageId · 에러사유`
→ 중복 발송 방지·정산·감사용. (예: "오늘 이미 보낸 사람" 체크)

## 4. 채널 전략 (알림톡 우선 + 문자 폴백)
- **알림톡**: 정보성 알림에 적합·저렴·스팸 차단 없음. 단 **사전 준비 필요**
  - 카카오 비즈니스 채널 개설 → Solapi에 **발신프로필(pfId)** 연결
  - **템플릿 사전 등록·검수**(영업일 소요): 변수는 `#{이름}` `#{완수율}` 형태
  - 정보성만 가능(광고 문구 불가), 버튼(과제 리포트 링크 등) 추가 가능
- **문자 폴백**: 알림톡 실패(미동의/미설치/검수 외 내용) 시 자동으로 LMS 발송
  - SMS 90바이트(한글 45자) / **LMS 2000바이트**(보호자 안내문은 보통 LMS)
  - **발신번호 사전등록 필수**(통신사 정책, Solapi 콘솔에서 본인 번호 등록)
- 구현: Solapi `send-many`의 알림톡 메시지에 **폴백(문자) 옵션**을 함께 실어 전송.

---

## 5. 발송 시나리오 (체크박스 선택형) + 메시지 초안
모든 시나리오는 **대상 자동 추출 규칙 + 템플릿**을 가진다. (변수 자동 치환)

| 시나리오 | 자동 대상 추출 | 메시지 초안 (변수 치환) |
|---|---|---|
| **① 미제출·미완 독려** | 최근 주차 완수율<100 또는 미제출 | "[오름] #{이름} 학생, 이번 주 과제(#{주차}) 완수율이 #{완수율}%입니다. #{기한}까지 마무리 부탁드립니다." |
| **② 가정통신문 요약** | 선택 수업 전체 | "[오름] #{이름} 학생 #{주차} 과제 결과를 안내드립니다. 평균 정답률 #{정답률}%, 완수율 #{완수율}%. 자세한 내용은 첨부 리포트를 확인해 주세요." |
| **③ 결석·지각 알림** | 교사가 당일 체크 | "[오름] #{이름} 학생이 금일 수업에 #{상태}하였음을 안내드립니다. 문의: #{선생님}" |
| **④ 직접 작성** | 수동 선택 | (자유 입력 + 변수 버튼 제공) |

- 미리보기에서 **글자수·SMS/LMS 구분·예상 단가·대상 수**를 실시간 표시.
- `[오름]`/`[가경T]` 브랜드 머리, 광고성 문구 배제(정보성 유지).

---

## 6. '문자 발송' 탭 UI 구성
1. **시나리오 선택** (라디오/체크) + 수업 필터 + 주차 선택
2. **대상 목록 테이블**: 체크박스 · 이름 · 번호(매칭/없음 경고) · 동의여부 · 완수율 등 근거 컬럼
   - "조건 대상 자동 선택" 버튼 (예: 미완자만 일괄 체크)
3. **메시지 편집기**: 템플릿 불러오기 + 변수 버튼(#{이름} 등) + 실시간 미리보기 + 글자수/단가/건수
4. **발송**: 확인 모달(대상 N명·예상비용) → 진행률 → 결과(성공/실패 카운트 + 실패자 목록 + 재시도)
5. **발송 로그** 하위 탭: 최근 발송 내역·결과 필터
6. **자동발송 설정**: 시나리오별 ON/OFF + 요일/시각 + 안전장치(동의자만/1일 1회/사전 미리보기)

---

## 7. 자동화 (수동 + 스케줄)
- **수동**: 위 UI 버튼으로 즉시 발송 (1단계 기본).
- **자동(스케줄)**: Apps Script **시간 기반 트리거**
  - 예) 매주 일 20:00 → '미완 독려' 대상 자동 추출 → 알림톡(폴백 문자) 발송 → 로그
  - **안전장치**: ①consent=Y만 ②동일 시나리오 24h 내 중복 발송 차단 ③1회 최대 인원 상한 ④발송 전 교사에게 요약 카톡/메일 통지(원하면 승인 후 발송 모드)

---

## 8. 개인정보·법적 체크
- 보호자 연락처 = 개인정보 → **수집·이용 동의(consent)** 관리, 미동의자 발송 금지.
- **정보성 메시지**(과제/성적/출결 안내)는 광고규제(야간발송 제한·080 수신거부) 대상 아님.
  단, 학원 홍보·모집 문구가 섞이면 **광고**로 분류 → (광고) 표기·수신거부·야간(21~08시) 발송 금지 적용.
- 발신번호 사전등록(통신사), 알림톡 템플릿 검수 통과 필수.

| 채널 | 대략 단가(건당) |
|---|---|
| 알림톡 | 7~15원 |
| SMS | 9~20원 |
| LMS | 30~45원 |
→ 알림톡 우선 정책이 **비용·도달률 모두 유리**. 선불 충전식.

## 10. 단계별 구축 로드맵
- **Phase 0 — 준비(교사)**: Solapi 가입·충전 / API 키 발급 / **발신번호 등록** / `명단` 시트 작성 / (알림톡용) 카카오 채널·**템플릿 검수 신청**
- **Phase 1 — 문자 수동 발송 MVP**: 명단 연동 → 대상 선택 → 미리보기 → LMS 발송 → 로그  *(알림톡 검수 기다리는 동안 먼저 가동)*
- **Phase 2 — 시나리오·변수 치환**: 미완 독려/가정통신문 요약 템플릿 + 자동 대상 추출
- **Phase 3 — 알림톡 + 폴백**: 검수 통과 후 알림톡 우선·문자 대체
- **Phase 4 — 스케줄 자동발송**: 시간 트리거 + 안전장치

---

## 11. 교사가 준비/확인해 줄 것 (다음 단계 전)
1. Solapi 계정 + **API Key / API Secret** (발급은 콘솔, 키는 내가 코드에 직접 안 넣고 Apps Script에 등록 방법 안내)
2. **발신번호 등록** 완료 여부 (등록한 번호)
3. 알림톡 진행 의향 시: 카카오 비즈니스 채널 유무 + 사용할 **템플릿 문구 확정**
4. `명단` 시트 양식대로 1개 수업만이라도 샘플 작성
5. Phase 1(문자)부터 먼저 만들지, 알림톡까지 한 번에 갈지
